    • creo sitios web

    modernos y conformes a las normas vigentes

    En cada proyecto posterior, trato de mejorar el código para hacerlo más legible. Tanto en HTML, CSS como JS, presto especial atención a las sangrías que mejoran la comodidad de trabajo y poner los nombres de los elementos semánticos del código. Mantengo el código dividido en secciones individuales, teniendo cuidado de no duplicarlo. CSS me permite crear cualquier apariencia de la página y JS decorarla, mejorarla o introducir interacción con el usuario.

    desarrollados para motores de búsqueda

    Diseño sitios web para que puedan ser encontrados en los primeros lugares de los buscadores. Para cada proyecto, dedico el tiempo suficiente en seleccionar palabras clave para que el posicionamiento final sea más que satisfactorio. Tampoco me olvido de los refrescos periódicos en el sitio web, gracias a los cuales los datos en mis sitios web siempre son actualizados.

    receptivo en todos los dispositivos móviles

    Escribo todos los proyectos de acuerdo con el principio de mobile first . Esto permite restaurar el sitio web sin ningún problema en todos los dispositivos móviles u otros dispositivos destinados a este fin. Estos no son solo dispositivos como computadoras portátiles o de escritorio, cuyos parámetros de pantallas / monitores están bastante bien establecidos. Pero también smartphones y tablets, que, debido a los constantes cambios de fabricantes, exigen constantemente estar al día.

    tecnologías que estoy aprendiendo

    Solo uso un marco de framework, porque acelera significativamente mi trabajo en el proyecto y es conveniente para introducir cualquier cambio. También puedo escribir sitios web sin necesidad de arranque.

    Uso GIMP para editar sin complicaciones los gráficos que necesito antes de ponerlos en la página. Me refiero, en primer lugar, a la edición de elementos como: iconos, banners, fondos, así como a la simple manipulación de fotografías.

    GitHub me permitirá trabajar tanto con texto como con lenguajes de programación. Da más control, proporcionando así muchas más posibilidades de gestionar los cambios en el documento. Y quizás lo más importante, GitHub tiene un historial de cambios para que los pasos de trabajo se puedan administrar más fácilmente.

    tecnologías que quiero aprender

    El uso de la tecnología Sass/Less mejorará y acelerará significativamente mi trabajo en proyectos más complejos. El mero uso de variables y el anidamiento de selectores facilita mucho el estilo.

    El uso de TypeScript ayudará a eliminar muchos errores triviales y difíciles de rastrear en el proceso del trabajo. TS contribuye a escribir código de mejor calidad y, por lo tanto, obtenemos un código transparente y autodocumentado.

    Para ampliar mis horizontes y ser más flexible, planeo aprender todos estos marcos. Cada marco tiene sus ventajas y desventajas, lo que significa que al desarrollar un proyecto, debe considerar cuál es el mejor para usar.

    Gulp me permitirá un mejor control sobre las actividades que se pueden realizar en js y css. GULP tiene una gran cantidad de tapones disponibles. Estos aditivos automatizan el trabajo y pueden mejorar nuestro producto final.
